# 💈 בארבר שופ — מדריך פריסה לאינטרנט (15 דקות)

בסוף המדריך יהיה לך **קישור אמיתי** (לדוגמה `https://barber-shop.vercel.app`) שעובד מכל טלפון — אנדרואיד ואייפון — עם בסיס נתונים משותף: כל תור שלקוח קובע נשמר ומופיע אצלך מיד.

---

## שלב 1 — יצירת בסיס הנתונים (Supabase, חינם)

1. היכנסו ל- https://supabase.com ולחצו **Start your project** (הרשמה עם חשבון Google).
2. לחצו **New project** → תנו שם (למשל `barber-shop`) → בחרו סיסמה חזקה ל-DB → Region: `Frankfurt (eu-central-1)` (הקרוב לישראל) → **Create**.
3. המתינו כדקה עד שהפרויקט מוכן.
4. בתפריט הצד לחצו על **SQL Editor** → **New query**.
5. פתחו את הקובץ `supabase/schema.sql` שבתיקייה הזו, העתיקו את **כל** התוכן שלו, הדביקו ולחצו **Run**. אמור להופיע `Success`.
6. בתפריט הצד: **Project Settings** → **API**. השאירו את הדף פתוח — תצטרכו שני ערכים:
   - **Project URL** (מתחיל ב-`https://`)
   - **anon public** key (מחרוזת ארוכה שמתחילה ב-`eyJ`)

## שלב 2 — העלאת הקוד ל-GitHub

1. היכנסו ל- https://github.com (צרו חשבון אם אין).
2. לחצו **New repository** → שם: `barber-app` → **Private** → **Create repository**.
3. בעמוד שנפתח לחצו **uploading an existing file** → גררו את **כל התוכן של התיקייה הזו** (לא את התיקייה עצמה — את הקבצים והתיקיות שבתוכה) → **Commit changes**.

## שלב 3 — פריסה ל-Vercel (חינם)

1. היכנסו ל- https://vercel.com → **Sign up** עם חשבון ה-GitHub שלכם.
2. לחצו **Add New → Project** → בחרו את `barber-app` מהרשימה → **Import**.
3. לפני הלחיצה על Deploy, פתחו את **Environment Variables** והוסיפו שניים:
   | Name | Value |
   |---|---|
   | `NEXT_PUBLIC_SUPABASE_URL` | ה-Project URL משלב 1 |
   | `NEXT_PUBLIC_SUPABASE_ANON_KEY` | ה-anon public key משלב 1 |
4. לחצו **Deploy** והמתינו כ-2 דקות.
5. זהו! תקבלו קישור בסגנון `https://barber-app-xxxx.vercel.app` — זה הקישור לשלוח ללקוחות.

## שלב 4 — בדיקה

1. פתחו את הקישור **בטלפון שלכם** → קבעו תור כלקוח.
2. פתחו את הקישור **במחשב** → כניסה כמנהל → התור מופיע ביומן. ✔
3. באנדרואיד: תפריט Chrome ⋮ → **הוספה למסך הבית** — האפליקציה תיראה כמו אפליקציה אמיתית עם האייקון שלה. באייפון: כפתור שיתוף → **הוסף למסך הבית**.

## דומיין משלכם (אופציונלי)

ב-Vercel: **Settings → Domains** → הוסיפו דומיין שקניתם (למשל `tor-barber.co.il`) ועקבו אחר ההוראות.

## ⚠️ שני דברים חשובים לדעת

**1. אבטחת המנהל.** בגרסה זו (MVP) בדיקת המייל של המנהל נעשית בצד הלקוח — זה מרתיע סקרנים אבל לא האקרים. לפני שמכניסים נתונים רגישים באמת, שלב ההקשחה הבא הוא Supabase Auth (התחברות Google אמיתית) עם מדיניות RLS שמגבילה כתיבה לטבלאות הניהול רק לחשבון שלך. התשתית מוכנה לזה.

**2. תזכורות וואטסאפ.** דורשות חיבור Twilio + אישור של Meta ל-WhatsApp Business API (תהליך של 1–3 שבועות — כדאי להתחיל אותו כבר עכשיו). בינתיים האפליקציה עובדת מלא בלי זה; מנגנון ההתראות בתוך האפליקציה פעיל.

```
├── lib/
│   ├── supabase.js            # חיבור ל-Supabase
│   └── usePersistentState.js  # שכבת הסנכרון: שמירה + realtime
├── supabase/
│   └── schema.sql     # סכמת בסיס הנתונים — להריץ פעם אחת

```
